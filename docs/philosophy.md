# Philosophy

## Context is infrastructure.

Not merely prompt data.

The same discipline applied to execution boundaries, routing contracts, and manifest topology must be applied to cognition lifecycle.

---

## The core problem

Operational agent loops become:

- **persistent cognitive processes** — they never terminate cleanly
- **context accumulators** — every cycle adds to the conversation
- **recursive reasoning organisms** — one investigation spawns another
- **background cognition sinks** — token cost grows, instruction fidelity decays

The system begins to think forever, remember everything forever, recursively explore forever.

This is not a model problem. It is an architecture problem.

---

## The boundary model

ContextLifecycleProtocol formalizes the same discipline applied elsewhere in the platform:

| System | What it enforces |
|--------|-----------------|
| Custodian | Architecture boundaries |
| OperationsCenter | Execution boundaries |
| RepoGraph | Visibility/projection boundaries |
| SwitchBoard | Routing boundaries |
| **ContextLifecycleProtocol** | **Cognition lifecycle boundaries** |

Boundaries without enforcement become documentation. ContextGuard is the enforcement.

---

## Loops should not think deeply

Loops should:

1. observe
2. classify
3. dispatch
4. checkpoint
5. summarize
6. sleep

The moment a loop begins architecture debates, repo archaeology, speculative redesign, or multi-hour debugging — it becomes an uncontrolled cognition sink. Deep work belongs to bounded disposable workers.

---

## Workers are disposable

A worker should:

1. load capsule
2. load only relevant evidence
3. perform bounded investigation
4. emit structured findings
5. update capsule
6. checkpoint
7. terminate

Workers must not become immortal sessions. The capsule survives the worker. The worker does not survive the lease.

---

## Artifact-first memory

**Persist:**
- evidence
- checkpoints
- summaries
- decisions
- artifacts
- provenance

**Do NOT persist as durable truth:**
- giant conversational histories
- speculative reasoning chains
- exploratory scratchpad cognition
- long-lived hidden summaries

The checkpoint is the memory. The conversation is the scratch pad.

---

## Context temperature

| Temperature | Contents | Workers should... |
|-------------|----------|-------------------|
| Hot | current blocker, active hypotheses, current lease | always load |
| Warm | recent remediations, current constraints, previous checkpoint | load when relevant |
| Cold | historical investigations, archived summaries | load only when explicitly needed |
| Frozen | historical logs, dead investigations, old archaeology | never load automatically |

---

## Schemas describe the boundary. ContextGuard enforces it. Checkpoints let the system survive without immortal sessions.

The vocabulary is not the system. The enforcement is the system.

---

## Relationship to Patterson's architecture

Andrew Patterson's [agent convention enforcement system](https://andrewpatterson.dev/posts/agent-convention-enforcement-system/) solves **convention drift** — agents that drift from documented conventions because conventions aren't enforced.

ContextLifecycleProtocol extends the same philosophy toward **cognition drift** — agents that drift from bounded, efficient operation because cognitive lifecycle isn't enforced.

The mechanism is the same: injectable context + enforcement hooks. The scope is different: ContextLifecycleProtocol governs the cognitive lifecycle itself, not just the conventions within it.
